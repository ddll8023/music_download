# 项目结构文档

## 1. 项目概述

- **项目名称**: QQ 音乐在线搜索与下载工具
- **项目类型**: 前后端分离的全栈 Web 应用
- **前端技术栈**: Vue.js 3 + Vite + Element Plus + Pinia + Vue Router（开发端口 3493）
- **后端技术栈**: Flask 3.1 + Flask-CORS（端口 3492）
- **项目定位**: 通过 QQ 音乐 API 实现歌曲搜索、在线播放与本地音乐扫描管理

## 5. 核心模块职责

### 5.1 前端核心

**`src/stores/Song.js`（Pinia Store）**
- 管理播放列表（playlist）
- 管理当前播放歌曲（currentSong）
- 管理播放状态（playing: boolean）
- 管理当前播放进度（currentTime）
- 管理搜索关键词与搜索结果

**`src/components/MusicPlayer.vue`**
- 接收 Pinia store 中的播放状态
- 实现播放/暂停、前进/后退、进度拖拽
- 显示当前歌曲信息（歌名、歌手、专辑图）

**`src/views/Home.vue`**
- 搜索框输入与搜索触发
- 展示搜索结果列表（歌名、歌手、专辑、时长）
- 点击歌曲触发播放，更新 store 中的 currentSong

**`src/utils/request.js`**
- Axios 实例，baseURL 指向后端 `http://localhost:3492`
- 请求拦截器：附加凭证 headers
- 响应拦截器：统一错误处理

### 5.2 后端核心

**`blueprint/song.py`（路由入口，前缀 /song）**
- `GET /song/search?keyword=xxx` - 搜索歌曲，返回歌曲列表
- `GET /song/albumImg?songmid=xxx` - 获取专辑封面 URL
- `GET /song/songUrl?songmid=xxx&songmid=xxx` - 获取歌曲真实播放链接

**`untils/search_song.py`**
- `search_song(keyword)` - 调用 QQ 音乐搜索 API
- `get_song_url(songmid_list)` - 批量获取歌曲播放链接（调用 QQ 音乐 v1 key 接口）
- `parse_playlist(url)` - 解析歌单 URL，返回歌单歌曲列表

**`credential/get_credential.py`**
- 读取 `credential/credential.json` 中的 uin、guid 等凭证
- 凭证过期时触发重新获取

**`common/result.py`**
- `Result.success(data)` - 成功响应 `{ code: 200, data, message }`
- `Result.error(message)` - 错误响应 `{ code: 500, data: null, message }`

## 7. API 路由汇总

> 基础 URL: `http://localhost:3492`
> 前端代理: Vite 配置将 `/api` 代理至 `http://localhost:3492`

### 7.1 歌曲相关（blueprint/song.py，前缀 /song）

| 方法 | 路由 | 参数 | 说明 |
|---|---|---|---|
| GET | `/song/search` | `keyword` (string) | 搜索歌曲，返回歌曲列表 |
| GET | `/song/albumImg` | `songmid` (string) | 获取指定歌曲的专辑封面 URL |
| GET | `/song/songUrl` | `songmid` (string, 可多个) | 批量获取歌曲真实播放链接 |

### 7.2 响应格式

成功响应:
```json
{
  "code": 200,
  "message": "success",
  "data": <实际数据>
}
```

错误响应:
```json
{
  "code": 500,
  "message": "<错误信息>",
  "data": null
}
```

## 8. 运行与部署

### 8.1 环境准备

**前置依赖:**
- Node.js（前端构建）
- Python 3.11+（后端运行）
- uv 或 pip（Python 包管理）

**Python 依赖安装:**
```bash
cd sys_python
pip install -r requirements.txt
# 或使用 uv
uv sync
```

**Node 依赖安装:**
```bash
cd sys_vue
npm install
```

### 8.2 开发启动

**后端启动（端口 3492）:**
```bash
cd sys_python
python app.py
# 或使用脚本
start_backend.bat
```

**前端启动（端口 3493）:**
```bash
cd sys_vue
npm run dev
# 或使用脚本
start_frontend.bat
```

启动后访问: `http://localhost:3493`

### 8.3 生产构建

```bash
cd sys_vue
npm run build
# 构建产物输出至 sys_vue/dist/
```

### 8.4 代理配置（vite.config.js）

Vite 开发服务器将 `/api` 请求代理至 `http://localhost:3492`，前端代码中无需写绝对地址:

```javascript
server: {
  port: 3493,
  proxy: {
    '/api': {
      target: 'http://localhost:3492',
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api/, '')
    }
  }
}
```

因此前端 request.js 中的 baseURL 应设为 `/api`，请求路径会自动映射到后端对应路由。
