# 音乐下载平台 - 完整部署指南

## 概述

本指南详细介绍音乐下载平台的完整部署流程，包括开发环境搭建、生产环境部署、配置说明和故障排除。

## 系统要求

### 基础环境

- **操作系统**: Windows 10/11, macOS 10.15+, Linux (Ubuntu 18.04+)
- **Python**: 3.7 或更高版本
- **Node.js**: 16.0 或更高版本
- **包管理器**: npm 或 yarn
- **Git**: 版本控制工具

### 硬件要求

- **最低配置**: 2GB RAM, 1GB 可用存储空间
- **推荐配置**: 4GB RAM, 2GB 可用存储空间
- **网络**: 稳定的互联网连接（用于获取音乐资源）

## QQ 音乐认证配置

### 获取认证信息

1. **打开浏览器**，访问 [QQ 音乐网页版](https://y.qq.com/)

2. **登录账号**，确保能正常访问音乐内容

3. **打开开发者工具**:
   - Chrome/Edge: F12 或右键 → 检查
   - Firefox: F12 或右键 → 检查元素

4. **获取 Cookie 信息**:
   - 切换到 "Network" (网络) 标签
   - 刷新页面或搜索一首歌曲
   - 找到任意请求，查看 Request Headers
   - 复制 Cookie 值中的关键字段

5. **创建认证文件**:
```bash
# 创建配置文件
cd sys_python/credential
vi credential.json
```

6. **配置文件格式**:
```json
{
  "qm_keyst": "你的qm_keyst值",
  "uin": "你的uin值", 
  "skey": "你的skey值",
  "qqmusic_key": "你的qqmusic_key值",
  "sessionid": "你的sessionid值"
}
```

### 验证认证有效性

```bash
cd sys_python
python -c "from credential.get_credential import judge_credential; judge_credential()"
```

## 生产环境部署

### 后端生产部署

#### 使用 Gunicorn（推荐）

```bash
# 安装 Gunicorn
pip install gunicorn

# 启动生产服务
gunicorn -w 4 -b 0.0.0.0:8080 --timeout 120 app:app

# 或使用配置文件
gunicorn -c gunicorn.conf.py app:app
```

**gunicorn.conf.py 示例**:
```python
bind = "0.0.0.0:8080"
workers = 4
worker_class = "sync"
timeout = 120
keepalive = 60
max_requests = 1000
max_requests_jitter = 100
user = "www-data"
group = "www-data"
```

#### 使用 Docker 部署

**Dockerfile** (sys_python/Dockerfile):
```dockerfile
FROM python:3.9-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    gcc \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 创建非 root 用户
RUN useradd -m -u 1000 appuser && chown -R appuser:appuser /app
USER appuser

EXPOSE 8080

CMD ["gunicorn", "-w", "4", "-b", "0.0.0.0:8080", "app:app"]
```

### 前端生产部署

#### 构建生产版本

```bash
cd sys_vue
npm run build
```

#### 使用 Nginx 部署

**nginx.conf 示例**:
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    # 前端静态文件
    location / {
        root /path/to/music_download/sys_vue/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
    
    # 后端 API 代理
    location /song/ {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

#### 使用 Docker 部署前端

**Dockerfile** (sys_vue/Dockerfile):
```dockerfile
# 构建阶段
FROM node:16-alpine as build-stage

WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

COPY . .
RUN npm run build

# 生产阶段
FROM nginx:alpine as production-stage

COPY --from=build-stage /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### Docker Compose 一键部署

**docker-compose.yml**:
```yaml
version: '3.8'

services:
  backend:
    build: ./sys_python
    ports:
      - "8080:8080"
    volumes:
      - ./sys_python/credential:/app/credential
      - ./sys_python/app.log:/app/app.log
    restart: unless-stopped
    
  frontend:
    build: ./sys_vue
    ports:
      - "80:80"
    depends_on:
      - backend
    restart: unless-stopped

  nginx:
    image: nginx:alpine
    ports:
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
      - ./ssl:/etc/nginx/ssl
    depends_on:
      - frontend
      - backend
    restart: unless-stopped
```

**启动命令**:
```bash
docker-compose up -d
```

## 监控和日志

### 后端日志配置

**日志轮转配置** (logrotate):
```bash
# /etc/logrotate.d/music-app
/path/to/music_download/sys_python/app.log {
    daily
    missingok
    rotate 30
    compress
    delaycompress
    notifempty
    create 0644 www-data www-data
    postrotate
        systemctl reload gunicorn
    endscript
}
```

### 系统服务配置

**systemd 服务文件** (/etc/systemd/system/music-backend.service):
```ini
[Unit]
Description=Music Download Platform Backend
After=network.target

[Service]
Type=exec
User=www-data
Group=www-data
WorkingDirectory=/path/to/music_download/sys_python
Environment=PATH=/path/to/music_download/sys_python/venv/bin
ExecStart=/path/to/music_download/sys_python/venv/bin/gunicorn -c gunicorn.conf.py app:app
ExecReload=/bin/kill -s HUP $MAINPID
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

**启用服务**:
```bash
sudo systemctl daemon-reload
sudo systemctl enable music-backend
sudo systemctl start music-backend
```

## 故障排除

### 常见问题

#### 1. 后端服务无法启动

**症状**: `python app.py` 报错或无响应

**排查步骤**:
```bash
# 检查 Python 版本
python --version

# 检查依赖安装
pip list | grep flask

# 检查端口占用
netstat -tlnp | grep 8080

# 查看详细错误
python app.py 2>&1 | tee debug.log
```

**解决方案**:
- 确保 Python 版本 >= 3.7
- 重新安装依赖: `pip install -r requirements.txt`
- 更换端口或终止占用进程
- 检查防火墙和网络配置

#### 2. QQ 音乐认证失效

**症状**: API 返回认证错误或 403 状态码

**排查步骤**:
```bash
# 验证认证文件
cat sys_python/credential/credential.json

# 测试认证
cd sys_python
python -c "from credential.get_credential import get_credential; print(get_credential())"
```

**解决方案**:
- 重新获取 QQ 音乐 Cookie
- 确保账号未被限制
- 检查 Cookie 格式是否正确

#### 3. 前端无法访问后端

**症状**: 网络请求失败，CORS 错误

**排查步骤**:
```bash
# 检查后端服务状态
curl http://localhost:8080/song/search -X POST -H "Content-Type: application/json"

# 检查 CORS 配置
grep -n "CORS" sys_python/app.py
```

**解决方案**:
- 确保后端服务正常运行
- 检查 CORS 配置中的允许域名
- 验证请求头和方法配置

#### 4. 音频文件无法播放

**症状**: 播放器显示错误或无声音

**排查步骤**:
- 检查浏览器控制台错误信息
- 验证音频文件 URL 是否可访问
- 测试不同浏览器和设备

**解决方案**:
- 检查音频文件格式支持
- 验证 HTTPS 环境下的安全策略
- 确保用户已进行交互激活音频

### 健康检查

```bash
#!/bin/bash
# health_check.sh

# 检查后端服务
if ! curl -f http://localhost:8080 > /dev/null 2>&1; then
    echo "后端服务异常，尝试重启..."
    sudo systemctl restart music-backend
fi

# 检查前端服务
if ! curl -f http://localhost > /dev/null 2>&1; then
    echo "前端服务异常，检查 Nginx..."
    sudo systemctl status nginx
fi
```
